# Illustration Guidelines

Illustrations inject warmth and humanity into the application, particularly during onboarding and empty states.

## Visual Style
- **Flat & Minimalist**: Avoid heavy 3D shading or skeuomorphism.
- **Abstract but Grounded**: Use geometric shapes combined with relatable human elements (e.g., a person sitting at a desk, a calendar floating with a checkmark).

## Color Rules
- **Dominant Colors**: White/Gray surfaces.
- **Accents**: Our core Indigo (`#4F46E5`) for primary objects and Emerald (`#10B981`) for success elements.
- **Shadows/Gradients**: Use only subtle, semi-transparent drop shadows (10% opacity) for depth. Avoid linear or radial gradients inside the illustration paths.
- **Corner Radius**: Match the UI (soft 8px to 16px rounding on abstract shapes).

## Categories
- **Human Illustrations**: Characters should lack specific facial features to remain universally relatable. 
- **Empty States**: Small, single-focus illustrations (e.g., a sleeping notebook for an empty journal).
- **Onboarding**: Larger, multi-element scenes demonstrating app value (e.g., tasks connecting to a calendar).

## Future Artwork
All marketing illustrations must strictly utilize this SVG-based, flat-vector approach to ensure they can scale perfectly to retina displays without artifacting.
