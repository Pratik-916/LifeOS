# Publication Assets & Legal Requirements

Prior to submitting the binaries to Apple and Google, the following external assets must be generated and hosted.

## Legal Documents
- **Privacy Policy**: Explaining how user data is handled. Because LifeOS features an Offline Sync Engine, emphasize that data is encrypted in transit and at rest.
- **End User License Agreement (EULA)**: Standard terms of service.
- **Hosting**: These must be hosted on a live URL (e.g., `https://lifeos.io/privacy`) that is publicly accessible.

## Support Infrastructure
- **Support Portal**: A way for users to report bugs or request account deletion.
- **Account Deletion Rule**: Apple strictly requires an in-app method to completely delete a user account and all associated data. This must be implemented and verified in the API.

## Content Ratings
- You will need to complete the IARC content rating questionnaire in the Play Console and App Store Connect, declaring that the app contains no restricted content.
