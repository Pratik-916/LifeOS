# Website Brand Guidelines

The LifeOS website (`lifeos.io`) is the primary marketing vehicle. It must strictly align with the mobile app's visual identity to ensure a seamless transition from landing page to download.

## 1. Landing Page (Home)
- **Hero Section**: High-contrast, massive typography (Inter Bold). A clean, isometric device mockup showing the Dashboard screenshot. The primary CTA must be the App Store and Google Play badges.
- **Feature Sections**: Alternating left-right zig-zag layout. Each section highlights one core pillar (Planner, Habits, Journal, Analytics) supported by the corresponding mobile screenshot.
- **Testimonials**: A scrolling marquee of user reviews. Clean, borderless cards with subtle Indigo accents.
- **Footer**: Dark Mode (`#111827`) contrasting the main page. Links to Privacy, Terms, Support, and FAQ.

## 2. Documentation & FAQ
- **Structure**: A sidebar-navigated knowledge base.
- **Typography**: Optimized for long-form reading (larger `line-height`, constrained `max-width: 65ch`).
- **Brand Consistency**: Uses the same Lucide/Heroicon icon set as the mobile app for category headings.

## 3. Blog
- **Hero Images**: Abstract vector illustrations mirroring the app's `illustration-style.md`.
- **Tone**: Insightful, focusing on actual productivity science rather than just app-updates.

## 4. Support & Contact
- Clean, single-column forms.
- Error messages on the contact form must mirror the app's forgiving, actionable tone (e.g., "Please include your email so we can reply" instead of "EMAIL REQUIRED").
